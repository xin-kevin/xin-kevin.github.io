Creating A Documentation Website

This document provides instructions on creating and deploying a document website in GitHub with Hugo on a Windows 7 computer.

Prerequisites

Before creating a document website, make sure the following conditions are met:

  • Hugo and Git are properly installed and configured on your computer.
  • A GitHub account is created.

Steps

1. Create a website with Hugo.

1.1 Open Command Prompt.

Click the Start button on the desktop, and then type cmd in the search box.

Expected outcome:
Command Prompt appears as follows.

1.2 Create a website by running the following command in Command Prompt:

hugo new site <website_name>
In the command above, <website_name> is the website to be created.

For example:
hugo new site website

Expected outcome:
Congratulations! Your new Hugo site is created in C:\Users\Administrator\website.

1.3 Download a Hugo theme.

  1. Open https://themes.gohugo.io/tags/documentation/ in an explorer, and then click the theme you are interested.
    For example, the Dot theme is chosen.
    The following procedure is described with the Dot theme as an example.

  2. Clone the Dot theme into the themes directory by running the clone command in Command Prompt:
    git clone git@github.com:themefisher/dot-hugo-documentation-theme.git themes/dot-hugo-documentation-theme

    Expected outcome:
    Cloning into 'themes/dot-hugo-documentation-theme'...
    ...
    Receiving objects: 100% (892/892), 4.56 MiB | 10.00 KiB/s, done.
    Resolving deltas: 100% (428/428), done.

  3. Go to the themes directory to check the Dot theme with the commands:
    cd themes
    dir
    You can find the Dot theme in the themes directory.
    For example:

1.4 Create a document in the website with the commands:

cd <website_name>
hugo new post/<document_name>.md

In the command above, <document_name> is the document to be created.
For example:
cd website
hugo new post/blog.md

Expected outcome:
C:\Users\Administrator\website\post\blog.md created

1.5 Edit this document.

Open blog.md with notepad++, and then add the following row above row draft: true:
type : "docs"

Then, you can edit blog.md by following the Markdown syntax.

1.6 Start the website server locally.

  1. Go back to the website root directory with the command:
    cd ..

  2. Start the website server locally with the command:
    hugo server -t dot-hugo-documentation-theme --buildDrafts
    In the command above, -t dot-hugo-documentation-theme specifies the Dot theme for the website.

    Expected outcome:
    Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)
    The URL of the web server is http://localhost:1313/.

  3. Visit the local website at URL http://localhost:1313/.
    Expected outcome:
    The webpage appears.
    For example:

2. Deploy the created website to GitHub.

2.1 Sign in to GitHub.

2.2 Create a new repository in GitHub.

  1. Click + and choose New repository from the menu.

    Expected outcome:
    The Create a new repository page appears.

  2. In the Create a new repository page, input the repository name in the Repository name field.

    Note:
    Make sure the repository name is <owner_name>.github.io, wherein <owner_name> is the value of the Owner field.

    For example:

  3. Click the Create repository button.
    Expected outcome:
    The new repository is created successfully.

2.3 Build a public website by running the following command in Command Prompt:

hugo --theme=dot-hugo-documentation-theme --baseUrl=https://<github_page> --buildDrafts
In the command above, --theme=dot-hugo-documentation-theme specifies the Dot theme for the website, and <github_page> is the name of the repository you created in GitHub.

For example:
hugo --theme=dot-hugo-documentation-theme --baseUrl=https://xin-kevin.github.io/ --buildDrafts

Expected outcome:
The website is built successfully and the public directory is created in the website root directory.

2.4 Upload the public directory to GitHub.

  1. Go to the public directory with the command:
    cd public

  2. Initialize the public directory as a Git repository with the command:
    git init

    Expected outcome:
    The public directory is initialized.

    For example:
    Initialized empty Git repository in C:/Users/Administrator/website/public/.git/

  3. Add files in the public directory to the local repository with the command:
    git add .

    Expected outcome:
    Files in the public directory are added to the local Git repository.

  4. Commit the files added in the local repository with the command:
    git commit -m "<commit_message>"
    In the command above, <commit_message> is the commit message.

    For example:
    git commit -m "first commit"

    Expected outcome:
    The files are committed and prepared to be pushed to the remote repository.

    For example:

  5. Add the remote repository for the local repository with the command:
    git remote add origin <Github_repository>.git
    In the command above, <Github_repository> is the URL of the GitHub repository created for deploying the website.

    For example:
    git remote add origin http://github.com/xin-kevin/xin-kevin.github.io.git

  6. Push the files in the local repository to the GitHub repository with the command:
    git push -u origin master
    Input your GitHub account name and password as required.

    Expected outcome:
    Files in the local repository are pushed to the GitHub repository.

  7. Verify if the website is available in GitHub.
    Visit the website at URL https://<github_page>, wherein <github_page> is the name of the repository you created in GitHub.

    For example:
    https://xin-kevin.github.io/

    Expected outcome:
    The webpage appears.
